        Implementasi Lexical Analyzer Berbasis DFA dan Algoritma C4.5

        Sistem ini dikembangkan sebagai tugas mata kuliah Pengantar Sains Komputasi. Menggabungkan Lexical Analyzer berbasis Deterministic Finite Automata (DFA) untuk tokenisasi dan validasi struktur teks, serta Algoritma Decision Tree C4.5 untuk klasifikasi kelayakan proposal berdasarkan 7 fitur linguistik dan akademik yang diekstrak secara otomatis.

        Lexical Analyzer dan DFA

        DFA (Deterministic Finite Automaton) memproses teks proposal karakter per karakter melalui state transitions yang deterministik untuk menghasilkan token stream.

        • 1
          Teks diproses token demi token menggunakan state machine
        • 2
          DFA berpindah state berdasarkan karakter input: huruf, angka, spasi
        • 3
          State akhir menentukan jenis token: KEYWORD, IDENTIFIER, NUMBER
        • 4
          Token stream dikumpulkan untuk analisis lanjutan C4.5

        Algoritma C4.5

        C4.5 membangun pohon keputusan menggunakan Information Gain Ratio. Fitur dengan entropy gain terbaik dipilih sebagai simpul pemisah keputusan.

        • 1
          Hitung entropy keseluruhan dataset proposal
        • 2
          Hitung information gain untuk setiap fitur akademik
        • 3
          Pilih fitur dengan gain ratio tertinggi sebagai pemisah
        • 4
          Rekursi hingga leaf node menghasilkan keputusan akhir

        7 Fitur Analisis

        Sistem mengekstrak 7 fitur kuantitatif dan kualitatif dari proposal untuk input klasifikasi C4.5:

        • F1
          Kepadatan dan rasio keyword akademik terhadap total token
        • F2
          Rasio term akademik versus identifier biasa
        • F3
          Kelengkapan bagian wajib: Latar Belakang, Tujuan, Metodologi, Hasil
        • F4
          Konsistensi data numerik dalam teks proposal
        • F5
          Ketiadaan token error (validitas leksikal)
        • F6
          Kekayaan kosakata (rasio identifier unik)
        • F7
          Panjang teks minimum yang memadai

        Tiga Kelas Output

        Sistem menghasilkan tiga kelas kelayakan berdasarkan skor akhir pohon keputusan C4.5:

        • L
          Layak — skor lebih dari 80%, memenuhi standar akademik
        • R
          Layak dengan Revisi — skor 55-79%, butuh perbaikan minor
        • T
          Tidak Layak — skor di bawah 55%, belum memenuhi kriteria minimum

        Dataset Training C4.5

        Berikut adalah 15 contoh proposal berlabel dari dataset training yang digunakan untuk menentukan bobot fitur pada algoritma C4.5. Setiap contoh menampilkan nilai 7 fitur yang diekstrak melalui Lexical Analyzer DFA beserta label kelas yang ditetapkan oleh evaluator akademik. Information Gain dari setiap fitur dihitung berdasarkan distribusi kelas pada dataset ini.

        ID Cuplikan Proposal F1 KwDens F2 AcRatio F3 Struct F4 Numeric F5 ErrRate F6 VocRich F7 Length Label

        Information Gain per Fitur

        Nilai information gain dihitung menggunakan rumus entropi Shannon terhadap distribusi kelas pada 15 sampel dataset di atas. Fitur dengan IG tertinggi dipilih sebagai simpul pemisah utama pada pohon keputusan C4.5.
